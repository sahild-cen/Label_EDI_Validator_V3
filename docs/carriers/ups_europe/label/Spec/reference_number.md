# Field: reference_number

## Display Name
Reference Number

## Field Description
Positions 11-17 of the Tracking Number, generated by the customer. The reference number is required to be unique for each package, similar to a progressive number used in the transportation industry.

## Format & Validation Rules
- **Data Type:** numeric
- **Length:** 7 digits (positions 11-17 of tracking number)
- **Pattern/Regex:** `[0-9]{7}`
- **Allowed Values:** Must be unique for each package
- **Required:** yes — embedded within tracking number

No examples in spec.

Embedded within the tracking number.

- **Typical Position:** Encoded within tracking number
- **Font / Size:** Not specified
- **Field Prefix:** None — part of tracking number data

## Edge Cases & Notes
- Must be unique per package — duplicate reference numbers within the same account are not permitted.
