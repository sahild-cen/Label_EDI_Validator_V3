What's New
New Features or Enhancements
What's New
Users shipping to Germany or from Germany will be
able see a new symbology on the shipping labels.
 If the user enters a weight that is greater than or equal 
10kg then a 10+ kg symbology will appear on the 
label. If the user enters a weight that is greater than or
equal to 20 kg, then the 20+kg symbology will appear
on the label. Examples of the labels can be found on 
page 52- 58.
Please note that all UPS labels featured in this guide 
are for visual representation only and should not be 
used as real shipping labels.
NOTE: Some label creation software providers and printer manufacturers have interpreted Mode 2 for U.S. destinations and Mode 3 
for non-U.S. destinations exclusively.

UPS Smart Label™ Design 
Introduction
Enabling Timely Deliveries and Accurate Information
UPS continues to apply sophisticated technology to 
computer-generated labels. The latest innovations to the 
UPS Smart Label™ design provides prompt deliveries, more 
accurate routing and 24/7 access to up-to-the-minute tracking 
and delivery information for all packages and shipments.
UPS aims to provide timely, accurate delivery information. 
By continuously investing in technology, UPS ensures that 
the most up-to-date routing and tracking methods are 
used as packages move from one location to another. These

 Label Segments
In an effort to make it easier for the label creator, the shipping label is divided into three separate parts called 
segments. Information can be changed within one segment without affecting the others. One segment is 
dedicated solely to the Carrier, one to the Receiver, and one to the Shipper (in that order, from top to bottom). 
These segments may be combined to form a single label or they can each comprise their own label. A shipping 
label, however, must include the Carrier segment, since that segment contains the information necessary 
for delivering the package. The Carrier segment must be applied to your package’s natural top, so consider 
this requirement when choosing your shipping label size.  The labels represented in this guide are 4" x 4.25" for 
regular labels and 4" x 6" and 4" x 8" when the extended area is used.

Label Certification Process 
• 
 Business Development or Customer Solution must provide the 
submission form and test labels to the UPS Europe Label Verification Team.
Submission Instructions                                                                                                     
Label certification is necessary when one of the following events occurs:
• 
 Customers must work directly with their UPS Contact at Business 
Development or Customer Solutions.

• 
 
The customer is new or the customer is implementing a new shipping 
application

• 
 
A new printer is being introduced
• 
 Physical labels must not be sent unless expressly agreed with Europe 
Label Verification e.g., in case the submission has been initiated because 
experiencing problems with existing labels by Operations. 

• 
 
New label stock is being introduced

• 
 
The customer is adding a new UPS Service
Submitting Labels

• 
 
Issues with currently approved labels have been identified
General Service Offerings (UPS® Standard, UPS Express, UPS Expedited, etc.)
Label Submission Requirements

• 
 
Submit a form for each customer location (multiple account numbers can 
be included if they are for the same customer name and address).
Electronic labels must be submitted to UPS Label Certification for final 
approval.  A full submission is required for each printer.  
Note: Don’t provide photo images for the certification process.

• 
 
Submit a minimum of one Pickup Summary Barcode report (if the 
application supports it).
• 
 Request electronic labels created using the Print to File or Print to PDF 
option. This label will be used by Customer Solutions to verify label 
requirements and operational functionality before submitting physical 
labels for approval. 
The label test set must be equally distributed among all printers supported. 
The number of required labels will be dependent on the number of 
supported UPS services. Please use the table below to determine how many 
labels should be submitted. 
Customers should not submit labels directly to Europe Label Verification. 
It is the responsibility of Business development and Customer Solutions 
to submit the labels and ensure the labels match the information on the 
submission form or it will delay the process.
     
• 
 Submit a minimum of 10 electronic labels, which were created by 
printing physical labels using the specified printer and scanning the 
labels. Please save this scanned label in PDF format. These labels will be 
used by Europe Label Verification to verify that the label retains label 
requirements and operational functionality as well as proper formatting 
when printed through the specified printer. This verification is required 
for a final label approval. 

• 
 Each of the labels above should be unique (i.e., a unique tracking number 
should be listed on each label).
If a customer must submit labels directly, it is the responsibility of the 
Customer Solutions Representative to receive images of the labels before 
they are submitted to ensure the labels match the information on the 
submission form. Any labels received in poor quality or not meeting 
requirements, will not be tested.

Definitions
AIM — 
Automatic Identification Manufacturers, 634 Alpha Drive, Pittsburgh, PA 15238-2802; a trade association 
of automatic identification suppliers.
Alphanumeric — 
A character set that contains alphabetic characters (letters), numeric digits (numbers) and usually other 
characters, such as punctuation marks.
Application Identifier — 
A character string which defines the specific intended use of the data that immediately follows.
Aspect Ratio — 
The ratio of barcode height to barcode length.
Bar — 
The dark element of a barcode, as opposed to a space between bars.
Barcode Block — 
Standardized unit of area in which bar-coded information is presented. barcode blocks are typically stacked 
vertically and are separated from each other by a horizontal line.
Barcode Character — 
A single group of bars and spaces that represents one number, letter, punctuation mark or other symbol. 
This is the smallest subset of a barcode’s data content.
Barcode Height — 
The bar dimension perpendicular to the bar (element) width.
Barcode Length — 
The physical length of a barcode symbol, excluding the quiet zones.
Barcode Symbol — 
An automatic identification technology that encodes information into an array of adjacent varying-width parallel 
bars and spaces.
Barcode Title — 
The name selected that describes the data content of the barcode symbol.
Bar Width — 
The thickness of a bar measured from the edge closest to the symbol start character to the trailing edge 
of the same bar.
Character — 
A single letter, number, punctuation mark or other special symbol.
Check Digit — 
A calculated character often included within a barcode symbol whose value is used for performing 
a mathematical check on the validity of the encoded data. (See also Modified MOD 10.)

Definitions
Code 128 — 
A variable length, continuous, self-checking, alphanumeric barcode. Its data character set contains 105 characters 
in each of three unique subsets. Each character consists of three bars and three spaces consuming 11 modules 
of width. Bars and spaces may be 1, 2, 3 or 4 modules wide. The name Code 128 is derived from the capability 
of encoding the full ASCII 128 character set. One of the three subsets (Code C) employs one Code 128 character 
for each character pair of digits from 00 to 99 permitting shorter barcode symbols in some situations where 
adjacent numeric characters appear. (See AIM USS 128 for symbology specifications.)
Coding Accuracy Support System (CASS) — 
A service offered by the United States Post Office to shippers and software vendors that improves the accuracy 
of matching addresses to ZIP+4™ codes. CASS provides a common platform to measure the quality of address-
matching software.
Compressed MaxiCode™ Data String — 
The process of reducing the number of bits necessary to represent characters that constitute a consignee 
address.
Data Identifier — 
A character string which defines the specific intended use of the data that immediately follows. The identifier 
is an alphabetic character preceded by up to three numeric characters as defined by ANSI MH10.8.2.
Decodability — 
A measure of printing accuracy as perceived by a barcode reader using a symbology reference decode algorithm.
Digit — 
Any of the numeric characters zero through nine, inclusive.
Domestic Movements — 
Domestic shipments are defined as package shipments that move within a single postal authority (most typically 
within a country). For example a Germany-to-Germany shipment would be considered a domestic shipment.
Edge-to-Edge — 
A measure from the leading edge of a bar or space to the next similar edge.
Element — 
A single bar or space in a barcode.
Font — 
A specific size and style of a printer’s type.
Highlighting Bars — 
One-tenth inch solid black lines located immediately above and below the UPS Barcode block.
Human-Readable Interpretation — 
The text representing the data content of a barcode.
Human-Readable Text — 
Refers to all text not representing encoded barcode data.
Intercharacter Gap — 
The space between the last bar of one character and the first bar of the next which separates the two 
adjacent characters.

Definitions
 
International Movements — 
International shipments are defined as package shipments that move between two or more postal authorities.
International Shipper’s Agreement — 
This document must be on file with UPS. Referred to in past Guides as the Power of Attorney.
Laser Printing — 
A print method in which an electrostatic image is used to transfer toner particles to paper, which is then fused 
by heat (similar to xerographic printing).
Uncompressed MaxiCode™ Data String (U.S. Destinations) — 
The postal code field for this Mode is a numeric field, which may contain 5 or 9 characters.  
For Example:  
Postal Code: 84170-6672  will be 841706672 in the MaxiCode™ data string i.e. <GS>96841706672<GS>
Postal Code: 28501will be 28501 in the MaxiCode™ data string i.e. <GS>9628501<GS>
NOTE: Postal code should not contain any spaces or special characters in the MaxiCode™ data string.  
The Mode 2 format for a compressed MaxiCode™ is not covered in this guide. 
***Note: Some label creation software providers and printer manufacturers have interpreted Mode 2 for U.S. 
destinations and Mode 3 for non-U.S. destinations exclusively.
Uncompressed MaxiCode™ Data String  (Non-U.S. Destinations) — 
Postal codes outside the US may be numeric/alphanumeric with a varying length. If the postal code is 6 characters long, 
the MaxiCode™ postal code field should be 6 CHARACTERS long.
For Example:  
Postal Code “A41460 ”
Postal Code field for MaxiCode Mode 3 “A41460 ”
If the postal code is MORE than 6 characters, TRUNCATE ANY CHARACTER OVER 6! 
For Example:  
Postal Code “TW137DY”
Postal Code field for MaxiCode Mode 3 “TW137D”
Postal code should not contain any spaces in MaxiCode unless used padding described above.
For Example:  
“W1T 1JY”
Should appear in the MaxiCode as “W1T1JY”
NOTE: Postal code should not contain any spaces or special characters in the MaxiCode™.  
The Mode 3 format for a compressed MaxiCode™ data string is not covered in this guide.
Mil — 
A dimensional unit equal to .001 inch, commonly used to refer to the width elements. (For Example:  .013 inch 
is equal to 13 mils.)
Minimum Reflectance Difference MRD — 
A comparison of the most reflective bar to the most absorptive space (i.e., the lightest bar and the darkest space).
Modified MOD 10 — 
The mathematically derived check digit that appears at the end of the UPS Tracking Number.
Modules — 
The narrow nominal bar or space in the barcode, typically referred to as the “X” dimension. Wider bars 
and spaces are often specified as multiples of one module.
Narrow Width — 
The nominal width dimension of the narrow bars and spaces in a barcode symbol. Also known as the “X” dimension.
Natural Top — 
The upper side when an item or carton is placed on its most natural, stable base.

Definitions
Nominal — 
The exact (or ideal) intended value for a specified parameter. Tolerances are specified as positive and negative 
deviations from this value.
Numeric — 
A character set that includes only numbers.
OCR Quality Fonts — 
Print settings readable by humans as well as machines. OCR (Optical Character Recognition) characters 
can be decoded by examining the vertical and horizontal axes.
Pitch — 
A measure from the leading edge of a bar or space of a character to the end of the character.
Print Contrast Signal (PCS) — 
A comparison between the reflectance of the bars and that of the spaces.
Print Quality — 
The measure of compliance of a barcode symbol with the requirements of accuracy, defects, modulation, 
symbol contrast and print technology.
Quiet Zone — 
An area containing no markings immediately preceding and following a barcode which may interfere with 
the scanning process (the areas above and below the barcode may also be designated as quiet zones).
Reference Number — 
Positions 11-17 of the Tracking Number generated by the customer. The reference number is required 
to be unique for each package (similar to a progressive number used in the transportation industry).
Separating Lines — 
The narrow lines used to segregate barcode and information blocks.
Service Icons — 
Black-and-white representations of the corresponding UPS services. 
Service Indicator — 
Positions 9-10 of the Tracking Number, indicating a package’s particular service features.
Space — 
The lighter element of a barcode usually formed by the background between the bars.
Start/Stop Characters — 
Distinct characters or patterns used at the beginning and end of each barcode symbol which provide 
the scanner with the start and stop reading instructions as well as the scanning direction indicator.
Subset A — 
One of the three unique subsets within Code 128 that enables the coding of Alphanumeric data. 
This subset is used at the beginning of the Tracking Number.
Subset C — 
One of the three unique subsets within Code 128 that enables the coding of two numbers within one Code 
128 character. This permits shorter barcode symbols in some situations where several adjacent numeric characters 
appear.

Definitions
 
Symbology — 
A set of rules for encoding information in a barcode symbol. An arrangement of parallel, rectangular bars 
and spaces arranged in a predetermined pattern following unambiguous rules.
Thermal Direct Printing — 
A print method in which a heated print head contacts specially treated labels, thereby causing the label to darken.
Thermal Transfer Printing — 
A print method in which a heated print head melts and transfers ink from a ribbon to a label.
Tolerance — 
The allowable variation from an ideal dimension.
Tracking Number — 
The UPS customized unique package ID when used with tracking/tracing services.
UPS Account Number — 
Positions 3-8 of the Tracking Number, containing the unique number assigned by UPS to the shipper.
UPS Routing Code (URC) — 
A representation of the delivery building for a package. The URC helps to ensure the accurate routing and prompt 
delivery of a shipment.
USPS CASS Database — 
The USPS CASS database is used to perform address validation for addresses in the United States.
USS-128 — 
Uniform Symbology Specification produced by AIM. These specifications are intended as a guide to aid 
the manufacturer, the consumer and the general public regarding printing and scanning Code 128.
Warsaw Agreement — 
The rules relating to liability established by the Convention for the Unification of Certain Rules Relating to 
International Carriage by Air signed at Warsaw, Poland, on October 12, 1929, and any amendments thereto 
shall apply to the international carriage of any shipment insofar as the same is governed thereby. There are no 
stopping places which are agreed upon at the time of tender of the shipment, and UPS reserves the right to route 
the shipment in any way UPS deems appropriate.
Wide-to-Narrow Ratio — 
The ratio between the widths of wide elements and narrow elements in two-width symbologies.
X Dimension — 
The nominal width dimension of the narrow bars and spaces in a barcode symbol.
ZIP Code™ — 
Five- or nine-digit number that identifies each postal delivery area within the United States.
ZIP+4™ Code — 
It is an enhanced version of the 5-digit ZIP Code­
. It consists of the 5-digit ZIP Code plus four additional digits 
that identify a specific range of delivery addresses.
ZIP Code and ZIP+4™ are registered trademarks of the United States Postal Service.
