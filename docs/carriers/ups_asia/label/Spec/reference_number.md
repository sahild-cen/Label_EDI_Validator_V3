# Field: reference_number

## Display Name
Reference Number

## Field Description
Positions 11-17 of the Tracking Number, generated by the customer. The reference number is required to be unique for each package (similar to a progressive number used in the transportation industry).

## Format & Validation Rules
- **Data Type:** alphanumeric
- **Length:** 7 characters (positions 11-17)
- **Pattern/Regex:** Not specified in spec
- **Allowed Values:** Not restricted — customer-generated
- **Required:** yes — must be unique for each package

## Examples from Spec
No examples in spec.

## Position on Label
Embedded within the Tracking Number (positions 11-17).

## Edge Cases & Notes
- Must be unique per package, functioning as a progressive/sequential identifier.
- This is a customer-generated component of the tracking number.
