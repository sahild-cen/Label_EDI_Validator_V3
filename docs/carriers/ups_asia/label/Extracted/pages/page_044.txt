TITLE/HEADER SECTION
The title/header must consist of the document title 
and vendor name, software and version number in 
the following format:
• Text “UPS Manifest — Summary Section”
• Left-justified in 12 pt. bold
The UPS Manifest Summary provides the shipper 
with a receipt for all packages accepted by UPS. 
Two copies of the UPS Manifest Summary must be 
printed after the successful upload of Package Level 
Detail (PLD). The UPS driver/agent will sign both 
copies and leave one copy with the shipper. If there 
are multiple PLD transmissions per day (staggered 
uploads), two copies of the UPS Manifest Summary 
must be printed for each PLD upload. Every upload 
must have a new transmission number. The UPS 
driver/agent must be provided with all UPS Manifest 
Summaries for this particular day. Specifications of 
font size are guidelines and should not supersede 
sound business decisions.
Vendor name, software name and version number 
centered in 12 pt. bold on the same line as the title
• For Example:  ABC Ltd. Ship Software v1.05
SEPARATOR LINE
Font Size = 1 pt. bold
SHIPPER INFORMATION SECTION
• Line 1 = Shipper Name
• Line 2 = Shipper Street Address
GENERAL RULES
The UPS Manifest Summary must be printed on 
blank white paper (no letterheads, logos or other 
form of printing). The UPS Manifest Summary 
must be printed from a computer printer in black 
ink. Handwritten corrections are not acceptable. 
Overlap of information into the next column is not 
permitted. The UPS Manifest Summary must begin 
on a new sheet of paper.
• Line 3 = 
Shipper Postal Code, City and Country 
Name
• Line 4 = 
Shipper Country Code (for country codes, 
please see Appendix E)
SHIPPING RECORD NUMBER CALCULATION
The Shipping Record Number is a 10-digit numeric 
field consisting of the seven-digit book number, 
the two-digit page number and the Modulo 7 check 
digit.
